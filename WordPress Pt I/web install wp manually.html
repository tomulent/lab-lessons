Doing it this way is not necessarily necessary, but it teaches you to be very self reliant. It's also crucial to know if you want to set up a WordPress development environment on you home computer using MAMP or WAMP. [youtube id="gsfzF0dGzB8" width="600" height="350" autoplay="no"] [youtube id="brOK5fi7_9Q" width="600" height="350" autoplay="no"] ## Step 1: Download WordPress We need to get an up to date copy of WP. Go to WordPress.org and follow the download links. It will download you a zip file that's about 5.5 MB in size. I like to keep this zip on my desktop or my WP development folder and give it a snazzy custom icon like . That way when you want to create a new instance of WP, then you can just double click the WP icon and get a fresh copy! I'm a huge nerd BTW. Just leave it there for now. ## Step 2: Create a New Database and User Login to your site's cPanel. If you forgot how to login, go to the email account with which you purchased your domain and search for the "New Account Information"  from Lithium Hosting. This has your Username and Password. You can't change your username (So far as we know. Figure out how to change it and get a prize.), but you can change your password easy enough. Refer to this post for instructions on how to do that. ### Step 2a: Click MySQL Database Wizard A database is like a bunch of spreadsheets. It's a carefully structured place for us to store large amounts of information, in this case, it will store settings and content for our WordPress site. MySQL (pronounced my sequel) is a language used to search through the database for information we want as well as making connections between points of data. SQL stands for Structured Query Language. And since I know you're worrying, the answer is no. No you won't be expected to know any MySQL. ### Step 2b: Create a Database Just write what you want this new database to be called. Just follow the naming convention of NO CAPITAL LETTERS and use underscores instead of spaces. My New WordPress = BAD my_new_wordpress = Good I titled mine "my_new_wordpress" but you might want something shorter. Less than 7 characters is ideal, that way you can make the database username the same thing. WRITE DOWN THE DATABASE NAME!!!   ### Step 2c: Create a User The database user is like a login to access the database. If you want multiple people to be able to view your database, then you would create multiple users. You don't want that though. Go ahead and create a username. Yes it does look weird that your cPanel username is prepended to the database name and database username. That's just how it's done. WRITE DOWN THE USERNAME AND PASSWORD!!! ### Step 2d: Give Yourself All Privileges This ties in with the whole "multipel users" thing. We need admin access to our database because we're admins. But you might want to limit the capabilities of other users. This is how you would do it. ## Step 3: Configure WordPress ### Step 3a: Title the Directory Unpack the WP install file you got from wordpress.org. Title the folder whatever you want to the permalink to this new instance to be. By that we mean, what do you type after "mysite.com" to get to this new WP installation we've made? Developers like naming things random stuff, so how about "donkey"? So eventually, you will type "www.mysite.com/donkey" into the URL bar to get to this WP site ### Step 3b: Set the Database Info Open up the WP folder and you'll see a whole bunch of files ending in php. Don't worry about all that right now. Just find the one called "wp-config-sample.php". Delete the "-sample" so it just says "wp-config.php". Then open it up. We're going to be changing some values on lines 19, 22, and 25. Read the comments and you'll notice that they tell you exactly what to do.   You can probably figure out what to do on you own here. You take all the information for the database and user you just created and insert it right where it says. ### Step 3c: Authentification You need to do some security stuff now. Don't worry it's easy. On lines 45-52 it tells you to "put your unique phrase here". On line 40 it gives you a link to a key generator at https://api.wordpress.org/secret-key/1.1/salt/. Visit that and copy everything all at once. ## Step 4: Upload to Your Server via FTP Using Cyberduck or FileZilla, upload your entire WordPress directory to your server. We do mean entire directory. If your using Cyberduck, just drag the directory into "public_html" which is your root. This may take some time since the directory is about 11MB in size. If something goes wrong during installation just hit try again, or whatever and tell it to overwrite ## Step 5: Run the Install Script In your browser, go to "www.yoursite.com/donkey/wp-admin/install.php ". You will then be taken to the install page. Go ahead and input all your information and you're done! That took slightly longer than using a script installer, but you actually learned something by doing it.
