How I  Saved A Lot of Time And Energy

When you signed up for the class we had you submit a writing sample, a survey, and a picture. It would be horrible to sort through if they were all in one folder so we wanted an individual folder for every person enrolled. There being upwards of 50 people, you can imagine how tedious that would be to create a folder for every single person. To get a folder in the format "Last Name, First Name" You would have to:
  1. Create a new folder
  2. Look at the roll
  3. Type out the name
  4. Repeat
That doesn't sound like much, but do it 50 times and it's pretty annoying. Why can't we do it all at once?

The Terminal

Your computer uses a Graphical User Interface (GUI) to make it analogous to a physical desktop complete with folders and files and such. In the olden days, you would do everything via a "Command Line Interface" (CLI). That means instead of double clicking on a folder called "30 A Students" to open it, you would need to write a command into the computer to make it perform that action. We won't get into all the commands that there are to execute, but suffice it to say, anytime you do anything on the computer (moving files, creating folders, deleting things, etc.) the computer is executing a command. Using a GUI is much faster and more intuitive for individual tasks, but what if you want to do the same thing over and over again, say, 50 times? Like make 50 folders?

A Comparison

Let's see what it looks like to make a folder using the GUI vs. the CLI

GUI

I start in a folder called "Example" and go to "File > New Folder" I've got a new folder, so I click on the title and rename it "Last Name, First Name" The whole thing took about 10 seconds. That's not painful, but that's only because I had to do it once. What if I had to do that 50 times? Don't forget you have to shift your focus between the folder and the list of names and believe me this takes a long, long time.

CLI

I start by going to "Applications > Utilities > Terminal.app" and opening it up This opens up the Terminal, Mac OSX's default Command Line Interface Working in the Terminal you need to be very specific. Using the GUI, you would just open up the folder "Example" and be "in" it. With Terminal, you need to tell it what folder it should be "in". To do this, you just type "cd" then a space, and then drag the folder "Example" onto the terminal window. Doing so will automatically enter the correct path to the folder. Hit Return and you're now "in" the folder "Example".  The command "cd" means "Change Directory" and the path that you supply it tells it exactly where to go. Now to make the folder, you would use the command mkdir "Last Name, First Name" As you can see, the name that we put in between quotes after the command "mkdir" creates a folder of that name.

Which Was Faster?

Probably the first one.  But that's what it was built for. Doing something once. What if I want to make 4 folders? I would just type the command "mkdir" followed by the name of the folders I wanted to create, in double quotes, separated by spaces. We now have the working knowledge to create 50 folders. We just need to manipulate the list we already have a bit to get it in this format.

Excel Functions

Naturally, our class roster comes to us as a spreadsheet so it's already in a format ripe for manipulation. Here's what it looks like (these are fake names of course) First name in column A, last name in column B. Remember, we want it in the format "Last Name, First Name". So we need to create a function that will do that.

Combine Values

We're going to make a function that will take the value from the cell in column B (last names), insert a comma and a space and then take the value from the cell in column A (first name). The simplest way to do that is with the following function, =B2 & ", " & A2 The "&" sign means "And". The comma and space in between double quotes means "text", that is, whatever text put between double quotes will show up as plain text. So, =B2 & ", " & A2  means, "take the value from cell B2 and a space followed by a comma, and the value of cell A2 and put them in that order in this cell. That's the format we want, for sure, but there's something else. Remember that we need to have each folder name in between double quotes. So we just need to update the function to put double quotes on either end. To do that you change the function to this,

=CHAR(34) & B2 & ", " & A2 & CHAR(34)

You might ask, "What's with the CHAR(34) thing? You said anything in between double quotes will show up in the cell." Well, that's what I thought too. One Last Bit of Formatting Left

We can't just paste this into the Terminal quite yet. Each name is on a different line, which Terminal will read as you actually hitting the return key and consequently will only make the first name and then give you a bunch of errors. So we need to clear the line breaks so that all the names are on one line and separated by a single space.

Paste the Column into Word as Unformatted Text

If you just do normal paste, it will be a table and we don't want that. Go to "Edit > Paste Special..." and select "Unformatted Text". You'll be left with something like this:

Open the Find and Replace Pane

The thing that we need to find is called a "Carriage Return". If you've ever seen an old timey typewriter, they had this lever that you would push to scroll the paper up a line and bring it back to the starting position so you could keep writing. [caption id="attachment_3633" align="aligncenter" width="500"] 7 is the carriage return lever[/caption] Basically when you hit the "Return" key, the computer inserts an invisible character called a "Carriage Return". We need to find all of these and replace them with a "Space". That would ultimately put everything on the same line. So how do you find them? Well you can't select them to copy paste into the find field, so we have to use a special character like we did to make the double quote a minute ago. Word has a little menu for special characters, but it's not very exhaustive, so I had to turn to Google to find it. The character code for Carriage Return in Word is ^13 Put that into the find field and just type a space into the replace field, hit Replace All and watch the magic happen. Pretty cool, eh?

Back To Terminal

Now it's just a matter of pasting that into the terminal after typing the "mkdir" command Hit return and let 'er rip.
